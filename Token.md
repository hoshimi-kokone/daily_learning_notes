# `Token`的概念

`Token`(又叫`词元`),它时大模型理解和生成文字的最小单位。

当你把一段文字输入给大模型,大模型并不能"理解"这些文字,而是通过工具`Tokenizer`(`分词器`)将文字切分为一个个`Token`,这些`Token`可能是:

1. 一个汉字: [鸡]
2. 两个汉字: [苹果]
3. 三个汉字: [孙悟空]
4. 一个标点: [。]
5. 一个单词: [apple]
6. 

# `Tokenizer`的工作流程

分词器的训练过程就是一个"找规律,建词表"的过程

## 第一步、统计高频组合

分词器通过分析大量文本,发现哪些字或字母经常出现,然后将它们打包成一个`Token`,并编号。

- [苹]、[果] 总是成双成对出现--->打包成一个`Token`,编号12345
- 英文中[i]、[n]、[g]经常连在一起出现--->打包成一个`Token`[ing],编号24680
- 逗号 [,] 超高频--->单独打包成一个`Token`,编号13579

## 第二步、构建词表

经过海量统计，`Tokenizer`会得到一个庞大的`Token`表,可能包含5万、10万，甚至更多`Token`。这个词表囊括了日常见到的各种字、词、符号。

## 第三步、转换和编码

当大模型工作时:

- 输入阶段: `Tokenizer`将文字切成`TOken`,转换成对应的数字编号
- 计算阶段: 大模型只需要处理这些数字
- 输出阶段: 分词器再把数字编号转换回人类能读懂的文字
